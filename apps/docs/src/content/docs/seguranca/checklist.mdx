---
title: Checklist de Segurança
description: Lista completa de verificações de segurança para deploy em produção.
---

## Base

- [ ] `.env` no `.gitignore`, `.env.example` com apenas valores placeholder
- [ ] HTTPS em todos os pontos (API Gateway + CloudFront + Vercel)
- [ ] Todos os inputs validados com modelos Pydantic
- [ ] Dependências auditadas (`pip audit` + `npm audit`)

## AWS / Serverless

- [ ] IAM roles com least privilege por Lambda (cada Lambda acessa apenas o necessário)
- [ ] Bucket S3 de resultados: leitura pública apenas via CloudFront OAC, escrita apenas pela Lambda de publish
- [ ] Validação de requisição API Gateway habilitada (JSON schema) antes do VTL mapping
- [ ] Throttling API Gateway configurado (50 req/s sustentado, 100 burst)
- [ ] CloudTrail habilitado
- [ ] Sem credenciais hardcoded em nenhum lugar (usar env vars / Secrets Manager)

## Integridade dos Dados Eleitorais

- [ ] Nunca aceitar BU sem verificação de assinatura (`sig_verified` flag)
- [ ] Deduplicação por chave composta: `{PROC}#{PLEI}#{TURN}` + `{UNFE}#{MUNI}#{ZONA}#{SECA}`
- [ ] Duplicata com assinatura diferente → flagrado para revisão manual
- [ ] Texto QR bruto armazenado para auditoria independente
- [ ] BUs imutáveis: insert-only (DynamoDB ConditionExpression)
- [ ] Cross-validation: `TOTC == NOMI + LEGC + BRAN + NULO`

## Frontend

- [ ] CORS restrito à origin do Vercel
- [ ] Nenhum dado sensível no código client-side ou localStorage
- [ ] CSP headers via configuração do Vercel
