## Premissas

- Período de ingestão: ~4 horas (urnas fecham às 17h, maioria dos BUs até 21h)
- Um BU por seção (sem retry por duplicatas)
- Tally recomputa a cada BU recebido (pode ser otimizado com batching)
- Lambda memory: 256 MB, duração média: 2s
- Step Functions Express: $0.000001 por transição de estado

## Otimizações Possíveis

1. **Batching no Tally**: acumular 10-50 BUs antes de recomputar reduz invocações em ~95%
2. **Lambda SnapStart** (Java) ou **provisioned concurrency**: irrelevante para Python com cold start < 500ms
3. **DynamoDB on-demand → provisioned**: só vale acima de ~200 WCU/s constantes
